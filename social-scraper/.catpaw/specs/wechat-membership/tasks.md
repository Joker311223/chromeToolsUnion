# 实施计划：微信会员系统

## 概述

本计划旨在为社交媒体抓取插件添加微信扫码登录和会员支付功能，并根据会员状态限制下载次数。我们将使用 TypeScript 进行前后端开发。

## 任务

- [ ] 1. 设置项目结构和核心接口
  - 创建后端目录结构并初始化 TypeScript 配置
  - 定义核心接口和类型（User, MembershipStatus, DownloadRecord）
  - 设置测试框架（Jest + FastCheck）
  - _需求: 4.1, 4.2_

- [ ] 2. 实现后端核心逻辑
  - [ ] 2.1 实现用户状态管理逻辑
    - 编写用户状态获取和更新逻辑
    - _需求: 1.3, 2.2_
  - [ ]* 2.2 编写用户状态属性测试
    - **属性 3: 月初重置**
    - **验证: 需求 3.4**
  - [ ] 2.3 实现下载限制逻辑
    - 编写下载次数检查和记录逻辑
    - _需求: 3.1, 3.2, 3.3_
  - [ ]* 2.4 编写下载限制属性测试
    - **属性 1: 下载限制**
    - **属性 2: 会员无限下载**
    - **验证: 需求 3.2, 2.3**

- [ ] 3. 检查点 - 确保所有后端测试通过
  - 确保所有测试通过，如有疑问请咨询用户。

- [ ] 4. 实现前端 UI 和交互
  - [ ] 4.1 更新 Popup UI
    - 添加用户中心、登录按钮和会员状态显示
    - _需求: 1.1, 1.3_
  - [ ] 4.2 实现二维码弹窗
    - 编写显示登录和支付二维码的逻辑
    - _需求: 1.1, 2.1_

- [ ] 5. 集成微信登录和支付
  - [ ] 5.1 实现微信登录流程
    - 集成微信扫码登录 API
    - _需求: 1.1, 1.2_
  - [ ] 5.2 实现微信支付流程
    - 集成微信扫码支付 API（模拟）
    - _需求: 2.1, 2.2_

- [ ] 6. 权限控制和下载限制集成
  - [ ] 6.1 在前端集成下载限制
    - 在导出 Excel 前检查下载权限
    - _需求: 3.1, 3.2_
  - [ ]* 6.2 编写集成测试
    - 测试端到端登录、支付和下载流程
    - _需求: 1.2, 2.2, 3.2_

- [ ] 7. 最终检查点 - 确保所有测试通过
  - 确保所有测试通过，如有疑问请咨询用户。
