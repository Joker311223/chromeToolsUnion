# 设计文档：微信会员系统

## 概述

本系统旨在为社交媒体抓取插件提供微信扫码登录和会员支付功能。通过会员身份验证，系统将对普通用户的下载次数进行限制（每月 2 次），而会员用户则享有无限次下载权限。

## 架构

系统由以下组件组成：
1. **Chrome 插件前端**：负责显示 UI、处理用户交互、发起登录和支付请求。
2. **后端服务 (Node.js/Express)**：负责处理微信 OAuth2 登录、支付回调、用户状态管理和下载计数。
3. **数据库 (Mock/SQLite)**：存储用户信息、会员状态和下载记录。

## 组件与接口

### 前端组件
- **Popup UI**：显示登录状态、会员信息和下载限制。
- **QR Code Modal**：显示微信登录和支付的二维码。

### 后端 API
- `GET /api/auth/wechat/qrcode`：生成微信登录二维码。
- `POST /api/auth/wechat/callback`：处理微信登录回调。
- `GET /api/user/status`：获取当前用户的会员状态和下载计数。
- `POST /api/payment/create`：创建微信支付订单并生成二维码。
- `POST /api/payment/success`：处理支付成功回调（模拟）。
- `POST /api/user/download`：检查并记录下载操作。

## 数据模型

### 用户 (User)
- `openid`: 微信唯一标识。
- `isMember`: 是否为会员。
- `downloadsThisMonth`: 本月已下载次数。
- `lastDownloadMonth`: 上次下载的月份（用于重置计数）。

## 正确性属性

*属性是系统在所有有效执行中都应保持的特征或行为——本质上是关于系统应该做什么的正式陈述。属性充当了人类可读规范与机器可验证正确性保证之间的桥梁。*

### 属性 1：下载限制
*对于任何* 普通用户，如果其本月下载次数已达到 2 次，则系统应拒绝其下载请求。
**验证：需求 3.2**

### 属性 2：会员无限下载
*对于任何* 会员用户，系统应始终允许其下载请求，无论其本月已下载多少次。
**验证：需求 2.3**

### 属性 3：月初重置
*对于任何* 用户，当当前月份与 `lastDownloadMonth` 不同时，系统应将 `downloadsThisMonth` 重置为 0。
**验证：需求 3.4**

## 错误处理

- **未授权**：如果用户未登录尝试敏感操作，系统应返回 401 错误。
- **达到限制**：如果普通用户达到下载限制，系统应返回 403 错误。
- **支付失败**：如果支付过程中出现错误，系统应提示用户重试。

## 测试策略

- **单元测试**：验证下载计数逻辑、月份重置逻辑和权限校验。
- **属性测试**：验证上述正确性属性在各种输入下的有效性。
